Overview

Base URL and general conventions.

ℹ️
All endpoints are prefixed with /api. All responses are JSON. Pagination uses page and pageSize query parameters. The API requires no authentication for read operations.
Base URL: /api
Format: JSON  |  Rate limit: 5 req/hour per IP  |  SSL: Required in production

Health

Service and database connectivity status.

GET
/api/health
Returns service health including database connectivity. Responds 200 (Healthy) or 503 (Unhealthy).
Try →
{ "status": "Healthy", "databaseConnected": true, "message": "Service is operational", "timestamp": "2025-01-01T12:00:00Z" }

Stats

Aggregated system-wide counts.

GET
/api/stats
Returns aggregated counts: total chefs, menu items, blog posts, testimonials, and subscriptions.
Try →
{ "totalChefs": 12, "totalMenuItems": 48, "totalBlogPosts": 8, "totalTestimonials": 24, "totalSubscriptions": 3 }

Menu

Categories, featured dishes, and full paginated menu.

GET
/api/menu/categories
List all menu categories.
Try →
GET
/api/menu/items/featured
List featured, currently-available menu items.
Try →
GET
/api/menu/items
Paginated list of all menu items with optional filtering.
Try →
GET
/api/menu/items/{id}
Get a single menu item by GUID with full details and chef info.

Query Parameters — /api/menu/items

ParameterTypeDefaultDescription
categoryIdGUID—optional Filter by category
searchstring—optional Full-text search term
pageinteger1optional Page number
pageSizeinteger10optional Items per page (max 100)

Error Codes

Standard HTTP status codes returned by the API.

StatusMeaning
200 OKRequest succeeded
400 Bad RequestValidation failed — check request body / query params
404 Not FoundResource does not exist
429 Too Many RequestsRate limit exceeded (5 req / hour per IP)
500 Internal Server ErrorUnexpected server error
503 Service UnavailableDatabase offline or health check failure
